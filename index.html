Foundations of Digital Media

Foundations of Digital Media

Project 1: Planned Obsolescence

The very first thing I learned in my first New Media course was how to use Adobe Photoshop. While I had some background in Photoshop from high school, I had never seriously considered it as a career choice until now. I quickly picked up both softwares. While the original images I created during this class are particularly poor by my standards today, I still value them as the places where I learned my fundamentals.

The first of these projects I created for this course was to create an image that communicated the idea of Planned Obsolescence. As we had a fair amount of old electronics in the house, I assembled a number of these old items into a pile and photographed them. I then used Photoshop to replace the blank screens within the image with advertisements for many of the modern counterparts of the items shown. While the image quality is relatively poor, it served as my first major project.

Project 2: Atlas Shrugged

My professor was a fan of the works of Ayn Rand. As such, our second assignment was a poster based on a quote from her literary work Atlas Shrugged. Once again, I developed this in Photoshop, but I approached this project using newly learned techniques such as the use of color adjustment and Photoshop's shading tools. This poster turned out incredibly well, and I am still happy with it today.

Project 3: Digital Privacy

While the original website is now gone, this was my first-ever foray into the world of website design and development. Being a first-year course taught by a professor who was not as familiar with web design as others, this website is by most standards quite poorly assembled, using multiple techniques that are often considered bad form (such as the use of table elements for spacing). Being my first attempt at creating a website however, it taught me the basics of how HTML is structured and put me on the path to furthering my career in web design. Pandemic Projects

Pandemic Projects

During the summer of 2020, I spent a fair amount of time out of work due to the COVID-19 pandemic, and began to experiment with programming again in my free time. After a few months of learning some of the more advanced concepts of Python as well as programmatically interacting with the Internet (via REST APIs and websockets), I created my first Discord bot. Named "Kaiser" at the suggestion of a friend, the bot could accept and respond to a basic set of commands regarding time-zone conversion. This experience proved fundamental to my initial understanding of how programs interacted with one another over the internet, and became a major part of my core skillset.

In hindsight, I made many mistakes with this bot, but also learned from them. While the use of .json to store all of the bot's data originally seemed like a good idea, in practice it proved slow with large data sets. In addition, my code lacked modularity, making it significantly harder to expand as the project got larger. However, it worked well for my first project, teaching me important skills about how data is retrieved and set around the world using the Internet. While the bot itself is no longer updated due to a lack of time on my part as well as the depreciation of a critical module, I have maintained its source code here.

Introduction to C++

Introduction to C++

While courses were being offered online, I took my first dedicated programming course. Using C++, I learned many fundamental concepts in this class, such as the use of compilers as well as Object-Oriented Programming techniques. The knowledge I gained in this class has been the foundations of my modern-day success with computer programming.

For the class's final project, we were required to design an interactive program using C++ to demonstrate the concepts we had learned. As I had been working in a restaurant at the time, I designed a command-line-based POS (Point-Of-Sale) terminal which could accept commands to create, modify, and complete orders with customized lists of items. While not as advanced as my earlier Python projects as well as my modern-day work, the code was significantly better formed in comparison, utilizing Object-Oriented patterns and a properly structured event loop. The command line was surprisingly robust and able to efficiently process multiple different commands. While I have since learned that my original code had security flaws, I still believe it shows how I had developed an understanding of important programming fundamentals, especially given that C++ is notorious for its level of difficulty.

If you would like to see the complete project, the entire project's code has been archived here.

Learning Flask and Creating Loonbooru

Learning Flask and Creating Loonbooru

The summer of 2022 would arguably be my most important summer ever for the advancement of my skills. Over the course of the summer, I decided to learn Flask, a Python framework for creating web servers that can serve dynamic content. While I was learning the library, a friend of mine came to me asking for help. He wanted a web site where he could upload and display images, but had been having trouble with existing services. Feeling up for a challenge, I constructed Loonbooru, my first properly open-source project. Over the course of the summer, I built this framework entirely from the ground up. While originally only able to serve pre-designated images, I have since added functionality to automatically generate browsing pages, a complex database-interface built entirely from scratch, a complex multi-threaded file uploading paradigm, and the ability to search for collections of images based on multiple parameters, including unique keywords. Loonbooru is easily my most ambitious project ever, and is still in active development.

While I will not provide a link to the original site Loonbooru was built for in respect of a request from the site owner, a link to the project's source code can be found here.
